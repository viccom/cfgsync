# cfgsync 用户指南

cfgsync 让你在多台设备之间同步软件配置。比如你的笔记本和台式机都用某个软件，配置一遍，两边就一样。

本文档面向最终用户与平台管理员（即运营服务的你+你的家人/朋友）。如果你想**开发接入 cfgsync 的软件客户端**，请看 [`developer-guide.md`](developer-guide.md)。

## 1. 一分钟理解

- **平台**：一个 Web 服务。你（用户）用浏览器访问；你使用的软件（客户端）用 HTTP API 访问。
- **配置**：一段文本（最多 4 MB）。你"拥有"它，存在云端。平台不解析内容，只负责存储和版本。
- **同步**：每台设备都能 GET（拉取）或 PUT（推送）配置。平台用"版本号"防止互相覆盖。

> **类比**：cfgsync 像是"软件配置的网盘"。但和网盘不同，平台不显示也不编辑你的配置内容——它只让软件自己读写。

## 2. 第一次使用

### 2.1 访问

向平台管理员询问服务地址（形如 `https://cfgsync.example.com`）。用现代浏览器打开。

### 2.2 注册

1. 点击"还没账号？去注册"。
2. 填邮箱（**作为登录账号**，不可更改）和密码（至少 8 位）。
3. 提交后自动登录。

> 家人/朋友使用场景下，任何人都能自助注册。管理员可以选择性提升某些用户为管理员。

### 2.3 登录

输入邮箱+密码即可。登录后默认跳到"我的应用"页。

## 3. 软件集成流程（核心）

这是普通用户唯一需要懂的流程。**核心一句话**：从 WebUI 申请一个 token，粘到软件里，软件就能自动同步。

```
浏览器（你）                  平台                 软件客户端
   │                           │                       │
   │ 1. 登录                    │                       │
   ├──────────────────────────►│                       │
   │ 2. 申请 app_token          │                       │
   ├──────────────────────────►│                       │
   │   返回 token（仅显示一次） │                       │
   │◄──────────────────────────┤                       │
   │ 3. 复制 token              │                       │
   │ 4. 粘到软件配置            │                       │
   │ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─►│
   │                           │ 5. 软件用 token 同步   │
   │                           │◄──────────────────────┤
```

### 详细步骤

1. **在 WebUI 里浏览"我的应用"**：看你想用的软件的 `app_id` 是否已被注册。如果列表里没有，请联系管理员注册。
2. **进入应用详情**：点应用的"管理 Token"。
3. **生成新 Token**：
   - 可选填标签（如"MacBook Air"）便于以后识别。
   - 点"生成新 Token"。
4. **立即复制 Token**（关键）：
   - 页面跳转到 `/show-token/...`，显示完整的 token 字符串（形如 `1rc_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`）。
   - **⚠️ 离开此页面后将永远无法再次看到该 token。**
   - 点"复制"按钮，或手动选中。
5. **粘贴到软件客户端**：找到软件设置里的"cfgsync / 配置同步 / cloud sync"之类的输入框，粘贴 token。
6. **完成**：软件会自动拉取/推送配置。你可以关掉浏览器。

> 如果以后丢了 token，重新申请即可。**新生成的 token 会立即让旧的失效**。

## 4. WebUI 各页面说明

### `/login`, `/register` — 登录与注册

切换 tab 即可。注册成功自动登录。

### `/apps` — 我的应用

显示所有平台已注册的应用（卡片列表）。每张卡片显示：
- 应用显示名（如 "1Remote Desktop"）
- 应用 `app_id`（灰色等宽字体，如 `com.1remote.desktop`）
- 描述

点"管理 Token"进入应用详情。

> 空状态："暂无可用应用。请联系管理员注册 app_id。"

### `/apps/{app_id}` — 应用详情（Token 管理）

这是你最常待的页面。

**上半部分：新建 Token**
- 标签输入（可选，但建议填，便于以后识别设备）
- "生成新 Token"按钮

**下半部分：我名下的 Token**
- 表格列出你为该 `app_id` 申请过的 token
- 每行显示：标签、前缀（如 `1rc_abcd1234…`，只显示前 12 位）、创建时间、最后使用时间、"撤销"按钮
- 撤销有内联二次确认

> 注意：**同一个 `(你, 应用)` 永远只有 1 个 token**。再次"生成新 Token"会让旧 token 失效。这是设计上的强约束，不是 bug。

### `/show-token/{app_id}/{token}` — 一次性 Token 展示

只在刚生成 token 时进入。**离开此页面后 token 无法找回**（服务端只存 SHA-256 hash，不存明文）。

页面提供：
- 复制按钮
- 醒目警告条
- `beforeunload` 事件钩子（关闭/刷新页面时弹窗提醒）

### `/me` — 我的配额

显示：
- **存储用量**：你所有应用的当前 payload 总字节，与上限（默认 100 MB）的百分比进度条。
- **App Token 数**：你已申请的 token 数量，与上限（默认 100）。

> 历史版本不计入配额（每 `(你, 应用)` 默认保留最近 50 次写入历史）。所以即使你写了 50 次 4 MB 的配置，占用是 4 MB，不是 200 MB。

### `/me/settings` — 设置

目前只有"退出登录"。

> 修改密码功能暂未提供。如需修改，请联系管理员（管理员可通过 SQL 直接重置 `password_hash`）。

### `/admin/apps` — 应用管理（仅管理员）

参见 [§5 管理员功能](#5-管理员功能)。

### `/admin/users` — 用户管理（仅管理员）

参见 [§5 管理员功能](#5-管理员功能)。

## 5. 管理员功能

管理员（`is_admin = true`）额外能访问两个页面。普通用户访问会被自动重定向。

### 5.1 应用管理 `/admin/apps`

表格列出所有 `app_id`：`app_id`、显示名、描述、创建时间、操作。

**新建应用**：
- `app_id`：**反域格式**（如 `com.example.myapp`），创建后不可改。规则：两个或更多点分段，每段 `[a-z0-9-]`，2-31 字符。
- 显示名：必填，≤ 256 字符。
- 描述：可选，≤ 1024 字符。

**编辑应用**：
- 可改显示名和描述。
- `app_id` 不可改。

**删除应用**（谨慎）：
- 会**级联删除所有用户**在该 `app_id` 下的所有数据（配置、历史、token）。
- 内联二次确认，但**无法撤销**。
- 删除前请通知所有相关用户。

### 5.2 用户管理 `/admin/users`

分页表格列出所有用户：邮箱、用户 ID（前 8 字符）、角色、创建时间。

操作：
- **提升为管理员**：点按钮即可，幂等（已是管理员的按钮变灰）。
- **降级**：当前版本不提供，需要直接 SQL（`UPDATE users SET is_admin=0 WHERE id=?`）。

> JWT 是无状态的：提升用户为管理员后，**用户当前的 access token 仍带旧的 `is_admin=false`**，需要重新登录或 `/auth/refresh` 才能生效（最多 1 小时，access token TTL）。

### 5.3 提升 / 重置密码（通过 SQL）

平台目前不提供"忘记密码"自助流程和"管理员重置密码"的 UI。如需手动操作：

```bash
# 进入服务器
ssh your-server
sudo -u cfgsync sqlite3 /opt/cfgsync/data/cloud.db

# 重置某用户密码（用 Go 跑 bcrypt cost=12 生成新 hash）
sqlite> UPDATE users SET password_hash='<新 bcrypt hash>', updated_at=strftime('%s','now') WHERE email='user@example.com';

# 提升管理员
sqlite> UPDATE users SET is_admin=1, updated_at=strftime('%s','now') WHERE email='user@example.com';

# 撤销某用户所有 refresh_token（强制重新登录）
sqlite> UPDATE refresh_tokens SET revoked_at=strftime('%s','now') WHERE user_id=(SELECT id FROM users WHERE email='user@example.com') AND revoked_at IS NULL;
```

> 生成 bcrypt hash 的小工具：`go run -mod=mod -tags=bcrypttool 'example.com/bcrypttool'`，或者任何在线 bcrypt 工具（注意 cost 用 12）。**不要用成本低于 12 的 hash**。

## 6. 概念速查

| 概念 | 说明 |
|---|---|
| **app_id** | 反域格式（`com.example.app`），标识"某软件的某数据格式契约"。管理员创建。 |
| **app_token** | 32 字符凭证（`1rc_xxx...`）。每个 `(你, 应用)` 一个，新生成让旧的失效。软件用它调 sync 接口。 |
| **payload** | 你存的配置文本（≤ 4 MB），平台不解析内容。 |
| **version** | 整数，每次 PUT 自动 +1。用于检测"别人/别设备改过"。 |
| **force** | PUT 时附 `?force=true` 跳过 version 检查。**慎用**，会覆盖别人的修改。软件一般不会自动用。 |
| **storage quota** | 你的所有应用当前 payload 总和，默认 ≤ 100 MB。 |
| **history** | 每次 PUT 都会留一份历史，最近 50 条（默认）。配额外的安全网。 |

## 7. 常见问题

### Q1：Token 不小心丢了/复制错了怎么办？

重新生成。WebUI → 应用详情 → "生成新 Token"。**旧 token 立即失效**，软件需要更新到新 token。

### Q2：忘了登录密码？

平台当前不支持自助找回。请联系管理员重置（参见 [§5.3](#53-提升--重置密码通过-sql)）。

### Q3：配额满了怎么办？

- 在 `/me` 查看具体哪个应用占得多。
- 进入对应应用详情，点"删除此应用下我的所有数据"（如果该应用你不再使用）。
- 或联系管理员临时提升你的 `USER_STORAGE_LIMIT_MB`。

### Q4：多台设备同时编辑，会发生什么？

平台用乐观锁。第二台设备 PUT 时会收到 `409 version_conflict`，响应包含 `current_*` 字段（云端当前内容）。**软件自己负责合并**——平台不会自动合并。

参见 [`developer-guide.md`](developer-guide.md) §"乐观锁协议"。

### Q5：软件报"配置已被其他设备更新"怎么办？

这是 Q4 的具体表现。看你的软件如何处理冲突——好的软件会：
1. 弹窗或后台拉取最新版本。
2. 让你选择保留哪边，或自动合并。
3. 再 PUT 一次。

如果软件没处理好，最简单的兜底是手动：在软件里点"重新拉取"或"强制覆盖"。

### Q6：能否同时给多个软件用同一个 token？

不能。每个 token 绑定一个 `(你, 应用)`。一个 token 不能跨 `app_id` 使用——平台会拒绝（返回 `403 forbidden`）。

如果你想为同一个软件在多台设备上同步，**所有设备共用同一个 token** 是支持的。

### Q7：能否为同一个应用申请多个 token（如一个笔记本、一个手机）？

不能。`(你, 应用)` 是 UNIQUE 约束，新生成让旧的失效。

如果你确实需要按设备区分，建议在标签里写明设备名（如"MacBook Air"），用同一个 token 跨设备——平台不在乎是哪台设备发的请求。

### Q8：管理员能看到我的配置内容吗？

管理员**不能通过 WebUI** 看你的 payload（WebUI 不渲染 payload）。但管理员**能**通过 SQL 直接查询数据库（即"信任管理员"模型）。如有隐私顾虑，请在 payload 里自行加密。

### Q9：JWT 过期后会发生什么？

access token 默认 1 小时过期。你的软件不直接用 JWT（用 app_token），所以软件不受影响。WebUI 在 JWT 过期时会自动用 refresh token（默认 30 天）续期；refresh token 也过期时需要重新登录。

### Q10：服务挂了/数据库损坏怎么办？

- 平台使用 SQLite + WAL 模式，且有每日 3:00 AM 自动备份（保留 30 天，路径 `/opt/cfgsync/backups/`）。
- 如果数据库损坏，从最近的备份恢复。详见 [`CLAUDE.md`](../CLAUDE.md) §"Deploy"。
